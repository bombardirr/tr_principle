Инструкция для агента Cursor на мини-ПК (деплой appzac / tr_principle)

Репозиторий: /home/ag/projects/tr_principle

Имена:
  - compose project:  appzac       (-p appzac)
  - app:  appzac-prod-app   (Go: API + SPA, порт 8080)
  - db:   appzac-prod-db
  - grafana: appzac-prod-grafana (proxy_net + localhost:3000)
  - prometheus: appzac-prod-prometheus (только 127.0.0.1:9090)
  - NPM Proxy Host → http://appzac-prod-app:8080
  - домен: appzac.ru
  - Grafana (NPM): grafana.appzac.ru

Подробности observability: deploy/observability/README.md

1) Один раз — окружение
   cd /home/ag/projects/tr_principle
   git clone git@github.com:bombardirr/tr_principle.git .   # если ещё нет
   cp .env.prod.example .env.prod
   # Заполните POSTGRES_*, DATABASE_URL, JWT_SECRET, CORS_ORIGIN,
   # METRICS_TOKEN, GRAFANA_ADMIN_*, GRAFANA_ROOT_URL.

2) Один раз — сеть NPM
   docker network create proxy_net || true

3) NPM (Nginx Proxy Manager) — приложение
   - Add Proxy Host: appzac.ru (или поддомен)
   - Forward Hostname / IP: appzac-prod-app
   - Forward Port: 8080   ← не 80, отдельного nginx-контейнера нет
   - SSL: Let's Encrypt + Force SSL / HSTS
   - Передавать X-Real-IP (по умолчанию NPM делает) — нужен для rate limit auth
   - Не публикуйте :8080 наружу без NPM

3b) Секреты
   - JWT_SECRET ≥ 32 символа (openssl rand -base64 48)
   - BACKUP_DIR=/data/backups (volume в compose)
   - CORS_ORIGIN = реальный origin SPA (https://appzac.ru)
   - METRICS_TOKEN (openssl rand -hex 32) — scrape /metrics
   - GRAFANA_ADMIN_USER=admin + GRAFANA_ADMIN_PASSWORD (openssl rand -base64 24)
   - GRAFANA_ROOT_URL=https://grafana.appzac.ru

3c) NPM — Grafana
   - Domain: grafana.appzac.ru
   - Forward: http://appzac-prod-grafana:3000
   - Websockets: on; SSL Let's Encrypt
   - Access List рекомендуется (иначе публична только форма логина; графики — после пароля Grafana)
   - Prometheus (:9090) на NPM не публиковать
   - Логин Grafana ≠ аккаунт appzac; пароль: grep GRAFANA_ADMIN_ .env.prod

4) Деплой вручную (порядок как в CI)
   cd /home/ag/projects/tr_principle
   git fetch origin main && git checkout main && git reset --hard origin/main
   docker compose --env-file .env.prod -p appzac -f docker-compose.prod.yml up -d db
   docker compose --env-file .env.prod -p appzac -f docker-compose.prod.yml up -d --build --remove-orphans
   # Старые appzac-prod-web / appzac-prod-api уйдут через --remove-orphans.
   # Также: node-exporter, prometheus, grafana.

5) Проверка
   docker exec appzac-prod-app wget -qO- http://127.0.0.1:8080/api/health
   docker compose --env-file .env.prod -p appzac -f docker-compose.prod.yml ps
   curl -s 'http://127.0.0.1:9090/api/v1/query?query=up'
   # Grafana: https://grafana.appzac.ru  (admin / GRAFANA_ADMIN_PASSWORD)
   # SPA: https://appzac.ru/ops/metrics (is_admin) → «Открыть Grafana»

6) Self-hosted runner — как раньше: /home/ag/actions-runner-appzac (не внутри репо).

7) CI
   Push в main → check → deploy: db → up --build app → health /api/health.
   .env.prod обязан содержать METRICS_TOKEN и GRAFANA_ADMIN_PASSWORD (не CHANGE_ME*).

Локально фронт: npm run dev (Vite), API: docker compose -f docker-compose.local.yml up.
